# AutoSense AI 项目调研报告

## 1. 项目概述

### 1.1 项目名称

AutoSense AI：面向车载智驾传感器行业的AI市场情报与产品方案工作台

### 1.2 一句话定位

AutoSense AI 是一个面向车载智能硬件、激光雷达和智驾方案团队的B端AI产品工具，帮助产品经理从公开行业信息、客户需求、竞品参数和研发约束中快速识别市场机会，输出产品定义、评估指标、售前方案和迭代建议。

### 1.3 项目目标

本项目不是做一个泛化AI助手，而是围绕智能硬件产品经理的真实工作流，构建一个垂直行业AI产品。项目目标包括：

1. 降低行业调研和竞品分析成本。
2. 提高客户需求拆解、产品定义和技术方案撰写效率。
3. 建立AI输出结果的评测体系，避免只依赖主观判断。
4. 打通“市场情报到产品机会，再到产品方案”的闭环。
5. 形成适合产品经理作品集展示的完整项目样例。

### 1.4 项目适配岗位能力

该项目对应产品经理岗位中的关键能力要求：

| 岗位要求 | 项目体现 |
|---|---|
| B端、C端产品策划 | 面向B端产品经理、售前、研发团队设计工作台 |
| 客户业务需求分析 | 通过客户需求录入、AI拆解和人工确认形成需求池 |
| 产品架构和功能需求设计 | 输出模块架构、功能清单、PRD和MVP范围 |
| 跟进研发、测试和上线 | 设计版本路线图、验收指标、迭代流程 |
| 行业动态和竞品分析 | 建立市场情报监测、竞品对比和机会评分 |
| AI产品全流程设计 | 覆盖场景挖掘、原型设计、MVP验证、效果评估 |
| AI评估体系 | 设计准确率、覆盖率、幻觉率、采纳率等指标 |
| A/B测试与数据驱动优化 | 设计用户行为数据、反馈采集和A/B测试方案 |
| 智能硬件行业理解 | 围绕激光雷达、车载智驾、车规可靠性和量产流程展开 |

## 2. 行业背景

### 2.1 智驾行业趋势

智能驾驶正在从基础辅助驾驶向高阶辅助驾驶发展。L2、L2+、城市NOA、高速NOA、自动泊车、L3有条件自动驾驶等功能逐渐成为主机厂差异化竞争的重要卖点。对车企而言，感知系统的稳定性、可靠性和成本控制直接影响智驾功能体验和量产节奏。

在这一背景下，激光雷达、毫米波雷达、摄像头、超声波雷达等传感器形成多传感器融合方案。激光雷达的优势在于能够提供较高精度的三维空间信息，尤其在远距离障碍物识别、复杂光照条件、异形障碍物识别等场景中有价值。但激光雷达也面临成本、车规可靠性、量产一致性、供应链稳定性和系统集成复杂度等挑战。

### 2.2 车载激光雷达市场特点

车载激光雷达行业具有以下特点：

1. 技术路线多样：机械式、半固态、MEMS、转镜、OPA、FMCW、1550nm、905nm等路线并存。
2. 应用场景分化：前向远距激光雷达、补盲激光雷达、舱外感知、Robotaxi和商用车场景的需求不同。
3. 客户定点周期长：车企定点涉及方案验证、样件测试、DV/PV验证、量产导入、质量体系和商务谈判。
4. 成本压力明显：规模量产后，车企对BOM成本、制造良率和供货稳定性要求更高。
5. 海外市场存在机会：海外车企在高阶智驾、法规合规、供应链多元化方面有持续需求。

### 2.3 产品经理在该行业的核心挑战

车载智能硬件产品经理不是只写需求文档，还需要在市场、客户、技术、研发、测试、供应链、售前和商务之间进行平衡。

典型挑战包括：

1. 行业信息分散，难以持续跟踪竞品动态和客户机会。
2. 客户需求表达不标准，常常混合业务目标、技术指标和商务约束。
3. 产品参数涉及硬件、算法、软件、结构、热设计、车规测试等多个领域。
4. 产品定义需要兼顾性能、成本、可靠性和量产可行性。
5. AI生成内容可能存在幻觉，需要建立可解释和可评估的工作流。
6. 售前方案和技术交流材料准备耗时，但又必须准确、专业、可追溯。

## 3. 市场机会分析

### 3.1 核心机会

AutoSense AI 的机会来自三个方面：

第一，智驾行业信息密度高，产品经理需要持续关注技术路线、车企动态、法规变化和竞品合作，但人工收集成本高。

第二，智能硬件产品定义复杂，产品经理需要把非结构化客户需求转化为可执行的研发需求、测试指标和售前方案。

第三，AI工具可以提升信息整理、文本生成、需求归纳和竞品对比效率，但现有通用AI工具缺少行业知识结构、评估体系和产品流程闭环。

### 3.2 目标市场

本项目的目标市场不是直接面向普通消费者，而是面向车载智能硬件和智驾产业链中的B端用户。

重点用户包括：

1. 激光雷达厂商产品团队。
2. 智驾域控、传感器融合和自动驾驶方案商。
3. 车企智驾部门、采购部门和前瞻技术部门。
4. 售前技术支持和海外市场团队。
5. 咨询、投资、行业研究和战略分析团队。

### 3.3 使用场景

| 场景 | 当前痛点 | AutoSense AI 价值 |
|---|---|---|
| 市场调研 | 信息来源分散，人工整理慢 | 自动聚合资讯、分类、摘要、机会评分 |
| 竞品分析 | 参数口径不统一，难比较 | 结构化竞品库和差异化分析 |
| 客户需求分析 | 客户表达模糊，需求混杂 | AI拆解业务需求、功能需求、性能指标和约束 |
| 产品定义 | 参数选择依赖经验 | 根据场景推荐产品方案和关键指标 |
| 研发协同 | 需求传递容易丢失上下文 | 形成可追溯需求池和版本计划 |
| 测试验收 | 标准不清晰，指标不完整 | 生成性能、可靠性、车规和场景测试指标 |
| 售前支持 | 材料准备耗时 | 自动生成技术方案摘要、PPT大纲和英文说明 |

## 4. 用户调研设计

### 4.1 用户画像

#### 用户A：智能硬件产品经理

核心任务：

1. 关注行业趋势和竞品动态。
2. 识别潜在客户机会。
3. 定义产品路线图和核心参数。
4. 协调研发、测试、供应链和销售资源。

痛点：

1. 信息收集耗时。
2. 技术参数多，难以快速形成产品判断。
3. 客户需求变化频繁，需求管理压力大。
4. 需要同时面向研发和客户讲清产品价值。

#### 用户B：售前技术经理

核心任务：

1. 理解客户场景。
2. 准备技术交流材料。
3. 对比自家产品与竞品方案。
4. 支持客户定点和商务推进。

痛点：

1. 售前材料需要频繁定制。
2. 海外客户需要英文材料。
3. 客户问题涉及技术、成本、量产和可靠性多个维度。
4. 信息更新快，材料容易过期。

#### 用户C：研发和测试负责人

核心任务：

1. 根据产品需求评估技术可行性。
2. 拆解开发任务。
3. 制定测试计划和验收标准。
4. 反馈需求变更带来的风险。

痛点：

1. 产品需求不够结构化。
2. 指标优先级不清楚。
3. 需求变更缺少影响评估。
4. 测试指标和业务目标之间缺少映射。

### 4.2 用户访谈提纲

为了让项目更像真实产品调研，可以设计如下访谈问题：

1. 你平时如何收集智驾和传感器行业信息？
2. 判断一个客户机会是否值得跟进时，你最关注哪些信息？
3. 产品定义阶段最难决策的参数是什么？
4. 你如何比较竞品产品？是否有统一模板？
5. 客户需求通常以什么形式传递给产品和研发？
6. 需求变更时，团队如何评估影响？
7. 售前方案和技术交流材料通常花费多少时间？
8. 你能接受AI辅助生成哪些内容？哪些内容必须人工确认？
9. 你认为AI输出的最大风险是什么？
10. 如果有一个行业情报和产品方案工具，你最希望它解决什么问题？

## 5. 竞品和替代方案分析

### 5.1 替代方案类型

AutoSense AI 的竞品不一定是同类软件，而是用户当前解决问题的替代方式：

1. 通用搜索引擎：百度、Google、Bing。
2. 行业资讯平台：汽车之家、盖世汽车、36氪、MarkLines、IHS/S&P Global等。
3. 通用AI工具：ChatGPT、Claude、Gemini、Kimi、通义千问、豆包等。
4. 协作文档工具：飞书、Notion、语雀、Confluence。
5. BI和数据分析工具：Tableau、Power BI、FineBI。
6. 内部知识库和销售CRM。

### 5.2 通用AI工具的不足

通用AI工具能提高文本处理效率，但在本项目场景中存在不足：

1. 缺少稳定的行业知识结构。
2. 难以持续跟踪特定竞品和客户。
3. 无法保证参数来源可追溯。
4. 缺少组织内部需求流转和权限管理。
5. 缺少AI输出质量评估体系。
6. 难以与产品路线图、研发任务和售前材料形成闭环。

### 5.3 差异化定位

AutoSense AI 的差异化不是“能聊天”，而是“能把市场信息转化为产品动作”。

差异化能力包括：

1. 垂直行业知识库：围绕激光雷达、智驾、车规、传感器参数构建信息结构。
2. 机会评分机制：对行业动态、客户事件和竞品变化进行可解释评分。
3. 产品定义模板：输出符合智能硬件产品经理工作方式的PRD、参数表、路线图和验收指标。
4. AI评测体系：持续监控AI输出准确性、完整性和人工采纳情况。
5. 售前场景支持：生成面向客户的方案摘要、英文材料和技术问答。

## 6. 产品方案设计

### 6.1 产品模块

AutoSense AI 包含六个核心模块：

1. 市场情报中心
2. 竞品知识库
3. 客户需求分析
4. 产品方案生成
5. AI评测中心
6. 售前材料生成

### 6.2 市场情报中心

功能目标：

1. 定期采集公开新闻、企业公告、车展信息、法规政策、招投标和竞品官网内容。
2. 对资讯进行分类、摘要、打标签和机会评分。
3. 输出“值得产品经理关注”的信息，而不是简单新闻列表。

情报类型：

| 类型 | 示例 | 产品意义 |
|---|---|---|
| 客户机会 | 某车企发布新一代智驾平台 | 可能产生传感器定点机会 |
| 竞品动态 | 竞品发布低成本补盲雷达 | 影响产品定价和参数规划 |
| 政策法规 | 自动驾驶法规更新 | 影响海外市场进入和合规测试 |
| 技术趋势 | FMCW或1550nm路线发展 | 影响中长期技术路线 |
| 商务事件 | 车企与供应商达成合作 | 判断竞争格局变化 |

机会评分维度：

1. 与目标市场相关度。
2. 与目标客户相关度。
3. 对产品路线图影响程度。
4. 时间紧迫性。
5. 信息来源可信度。
6. 商业转化可能性。

### 6.3 竞品知识库

核心字段：

| 字段 | 说明 |
|---|---|
| 厂商名称 | Hesai、RoboSense、Luminar、Innoviz等 |
| 产品型号 | 具体型号 |
| 技术路线 | 905nm、1550nm、MEMS、转镜、FMCW等 |
| 探测距离 | 不同反射率下的距离 |
| 视场角 | 水平和垂直FOV |
| 点频 | 每秒点数 |
| 帧率 | Hz |
| 分辨率 | 角分辨率 |
| 功耗 | W |
| 防护等级 | IP等级 |
| 工作温度 | 摄氏度范围 |
| 量产状态 | 样机、定点、量产 |
| 客户案例 | 已公开客户 |
| 信息来源 | 官网、公告、新闻、财报等 |
| 更新时间 | 最近更新日期 |

### 6.4 客户需求分析

客户输入示例：

“欧洲某车企计划在2027年量产L3车型，需要一款适配高速NOA和拥堵自动驾驶场景的前向激光雷达，要求远距离探测、低功耗、满足车规可靠性，并希望供应商提供英文技术材料和测试报告。”

AI拆解结果：

| 需求类别 | 拆解结果 |
|---|---|
| 业务目标 | 2027年量产L3车型 |
| 应用场景 | 高速NOA、拥堵自动驾驶 |
| 产品形态 | 前向长距激光雷达 |
| 性能需求 | 远距离探测、高分辨率、稳定帧率 |
| 工程约束 | 低功耗、易集成、车规可靠性 |
| 合规要求 | 海外法规、功能安全、可靠性测试 |
| 交付物 | 技术方案、英文材料、测试报告 |
| 风险点 | 开发周期、成本、认证、竞品定点 |

### 6.5 产品方案生成

系统根据客户需求、竞品数据和产品策略生成产品定义草案。

输出内容包括：

1. 产品定位。
2. 目标客户。
3. 应用场景。
4. 核心卖点。
5. 关键参数。
6. 研发需求。
7. 测试验收指标。
8. 商务和售前建议。
9. 风险与待确认问题。

示例产品定义：

## 7. AI产品设计

### 7.1 AI能力边界

AI在本项目中的角色是“辅助分析和生成”，不是完全自动决策。

AI可以做：

1. 信息摘要。
2. 分类打标。
3. 需求拆解。
4. 竞品参数抽取。
5. 机会评分解释。
6. 文档初稿生成。
7. 测试指标建议。
8. 售前问答草稿。

AI不应直接做：

1. 未经验证的产品参数承诺。
2. 未经人工确认的客户报价。
3. 未经法务和质量确认的合规结论。
4. 未经研发评审的技术路线决策。
5. 自动对外发送正式材料。

### 7.2 AI评估指标

| 指标 | 定义 | 评估方式 |
|---|---|---|
| 需求拆解准确率 | AI是否正确识别客户核心需求 | 与人工标注答案对比 |
| 信息覆盖率 | 输出是否覆盖性能、成本、可靠性、量产、合规等维度 | 维度检查表 |
| 来源可追溯率 | 关键结论是否有来源 | 统计带来源结论占比 |
| 幻觉率 | 是否出现无依据参数或错误事实 | 人工审核和事实核验 |
| 方案匹配度 | 推荐方案是否匹配场景 | 专家评分 |
| 人工采纳率 | 用户是否采用AI建议 | 编辑记录和采纳按钮 |
| 时间节省 | 相比人工完成节省时间 | 用户任务计时 |
| 用户满意度 | 用户对输出质量的评分 | 任务后评分 |

### 7.3 测评数据集设计

MVP阶段可设计一个小型数据集：

1. 30条行业资讯样本。
2. 20条客户需求样本。
3. 10个竞品产品样本。
4. 5类典型应用场景。
5. 3套人工标准答案。

样本类型：

| 样本类型 | 数量 | 用途 |
|---|---:|---|
| 新闻资讯 | 30 | 测试分类、摘要、机会评分 |
| 客户需求 | 20 | 测试需求拆解和方案推荐 |
| 竞品参数 | 10 | 测试参数抽取和对比 |
| 售前问题 | 20 | 测试问答生成 |
| 验收场景 | 10 | 测试指标生成 |

## 8. MVP范围

### 8.1 MVP目标

MVP不追求全自动实时系统，而是验证三个核心假设：

1. AI能否有效降低行业调研和需求拆解成本。
2. 结构化竞品库能否帮助产品经理更快形成判断。
3. AI输出是否能通过评测体系持续优化。

### 8.2 MVP功能范围

P0功能：

1. 行业资讯列表和AI摘要。
2. 情报分类和机会评分。
3. 客户需求输入和AI拆解。
4. 竞品参数对比表。
5. 产品方案草案生成。
6. AI输出评分和人工反馈。

P1功能：

1. 售前方案摘要生成。
2. 英文技术介绍生成。
3. 产品路线图管理。
4. 需求变更影响分析。

暂不做：

1. 自动爬取全网。
2. 自动生成正式报价。
3. 自动发送客户材料。
4. 与PLM、CRM、Jira等系统深度集成。

## 9. 商业价值

### 9.1 对产品团队

1. 缩短行业调研周期。
2. 提升需求拆解质量。
3. 提高产品定义文档标准化程度。
4. 支持产品路线图决策。

### 9.2 对售前团队

1. 快速生成客户定制化方案。
2. 降低重复材料撰写成本。
3. 提高海外客户技术交流效率。
4. 保持材料与最新竞品信息同步。

### 9.3 对管理层

1. 更快识别市场机会。
2. 更清晰掌握竞品威胁。
3. 更好评估产品立项优先级。
4. 形成数据驱动的产品决策机制。

## 10. 风险分析

| 风险 | 表现 | 应对策略 |
|---|---|---|
| 信息准确性风险 | 抓取信息过期或错误 | 标注来源、更新时间和可信度 |
| AI幻觉风险 | 生成不存在的参数或结论 | RAG检索增强、关键参数强制引用来源 |
| 行业知识不足 | AI无法理解专业语义 | 建立领域词库和模板 |
| 用户信任不足 | 用户不敢采纳AI输出 | 提供人工确认、评分和修改记录 |
| 数据合规风险 | 抓取或使用非授权数据 | 优先使用公开来源和合规API |
| 产品过大风险 | MVP范围失控 | 聚焦情报、需求、竞品、方案四个核心链路 |
