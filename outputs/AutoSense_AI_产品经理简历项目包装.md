# AutoSense AI：车载激光雷达市场机会与产品决策工作台

GitHub 链接：https://github.com/MwngxuZhang/autosense-ai

技术栈：Python、SQLite、Vanilla JavaScript、DeepSeek API、RSS/搜索抓取、RAG 资料依据库、Webhook、Render 动态部署

## 项目概述

独立设计并落地一款面向车载激光雷达产品团队的 B 端 AI 产品决策工作台，服务产品经理、战略市场、售前方案和研发协同场景。系统支持公开市场情报抓取、机会评分、客户需求拆解、竞品 Battlecard、产品方案生成、资料依据库问答、AI 效果评估和协同任务推送，形成从“市场信号”到“产品动作”的闭环。

项目以 RoboSense 速腾聚创业务场景为参考，围绕 EM4、EMX、E1、M1 Plus、P6 等产品线，以及海外 L2+/L3 乘用车、Robotaxi 等目标客户画像，搭建可维护、可扩展的产品决策流程。

## 业务背景

车载智驾行业信息变化快，产品经理需要持续关注客户定点、竞品发布、法规趋势、传感器技术路线和海外市场机会。但相关信息分散在新闻、官网、客户沟通、内部资料和售前反馈中，直接转化为产品需求和方案的效率较低。

传统信息流工具只能提供新闻聚合或摘要，难以回答产品经理真正关心的问题：这是不是机会，匹配哪条产品线，客户信息缺什么，竞品威胁是什么，研发和售前下一步该做什么。AutoSense AI 的定位是把外部市场信号转化为可追溯、可协同、可落地的产品判断。

## 主要工作

- 负责产品定位和场景设计，明确核心用户为车载激光雷达产品经理、战略市场、售前方案和研发协同团队，梳理“市场监测、机会判断、需求拆解、方案生成、协同推进”的主流程。
- 设计自动市场雷达模块，支持公开 RSS/搜索源抓取，对新闻和市场线索进行摘要、分类、标签化、机会评分和产品线匹配。
- 设计需求拆解模块，将客户表达或市场机会拆解为应用场景、性能指标、可靠性要求、交付物、风险点和待确认问题，降低需求沟通成本。
- 设计竞品对比和 Battlecard 模块，围绕 Hesai、Luminar、Innoviz、Aeva 等竞品沉淀技术路线、公开客户、威胁判断和售前应答材料。
- 设计产品方案生成模块，基于我方产品画像、客户需求、竞品证据和资料依据库，输出产品定位、目标场景、关键规格、研发任务、验收标准和风险提示。
- 设计资料依据库，将产品手册、测试说明、客户纪要和方案备忘录作为 AI 回答依据，提升结果可追溯性，降低幻觉风险。
- 设计产品画像维护能力，支持产品线和目标客户画像的模块化新增、删除、保存，确保系统先理解“我方产品”和“目标客户”再分析市场机会。
- 设计 AI 效果评估体系，围绕准确性、覆盖率、可追溯性、幻觉风险和人工复核结论记录模型输出质量。
- 完成动态 Web 应用实现，支持用户在浏览器内配置自己的 DeepSeek API Key；Key 不提交仓库，部署后体验者可自行配置。
- 补充 Render 动态部署配置、GitHub README、页面预览图和项目文档，使项目可展示、可运行、可解释。

## 项目成果

- 落地一套可运行的 AI 产品工作台，覆盖市场情报、需求分析、竞品对比、产品方案、资料依据库、AI 评测和协同推进等核心模块。
- 实现从外部市场信号到产品方案草案的闭环：自动抓取线索，形成机会评分，生成需求拆解，再输出产品方案和协同动作。
- 支持用户自带 DeepSeek API Key，满足公开项目展示场景下的安全配置需求，避免在 GitHub 暴露真实密钥。
- 完成产品线和目标客户画像的模块化维护，能够解释产品方案“基于什么生成”：基于客户需求、我方产品能力、竞品证据和资料依据库。
- 输出 P0/P1/P2 能力规划，覆盖 MVP 可用能力、增强能力和生产化能力，为后续迭代提供清晰路线。
- 形成可放入 GitHub 的动态部署项目，包含部署配置、README、页面预览图和简历表达材料。

## 面试讲述版

这个项目不是单纯做新闻摘要，而是解决车载激光雷达产品经理的真实工作问题：外部信息很多，但难以直接转成需求、方案和研发动作。我把产品定位为“市场机会与产品决策工作台”，核心用户是产品经理、战略市场、售前方案和研发协同团队。

设计上我参考了 AlphaSense 的证据链结论、Klue 的竞品作战卡、Productboard 的需求管理、Aha! 的路线图协同和 Feedly 的情报订阅，但没有照搬，而是垂直落到车载激光雷达场景。系统先维护我方产品线和目标客户画像，再抓取市场机会，最后结合客户需求、竞品证据和资料依据库生成产品方案。

如果面试官问这个项目的特殊点，可以概括为三点：第一，它不是信息流，而是产品决策闭环；第二，它不是让 AI 凭空生成方案，而是基于我方产品画像、客户字段、竞品证据和资料依据库生成；第三，它考虑了真实公开展示和运行安全，支持用户自己配置 API Key，不把密钥放进仓库。
