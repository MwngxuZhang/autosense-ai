# AutoSense AI P0/P1/P2 能力补齐技术文档

## 1. 改造目标

本次改造基于《AutoSense AI 类比竞品调研报告》的 P0/P1/P2 功能清单，把页面从“接口展示/参数表展示”升级为真实产品经理可使用的产品工作台。

核心原则：

1. 网站上不出现 `parameter/value`、原始 JSON、大段接口返回等调试型表达。
2. P0/P1/P2 不只写在文档里，而要在网页中形成可浏览、可解释、可追踪的能力路线图。
3. 每个能力都要说明来源借鉴、业务价值、当前状态和下一步动作。
4. 面向产品经理、售前、研发协同和管理层汇报，而不是面向开发调试。

## 2. P0 能力补齐

P0 是当前系统必须优先具备的能力，目标是证明产品不是普通 AI 摘要工具。

| 能力 | 借鉴对象 | 网页表现 | 当前实现 |
|---|---|---|---|
| 引用型 AI 结论 | AlphaSense | 证据链市场结论 | 已实现 |
| 高机会事件中心 | Dataminr | 高机会池、机会评分、推送阈值 | 已实现 |
| 客户信息完整性检查 | Productboard/Klue | 完整性进度、缺失字段、待确认问题 | 已实现 |
| 竞品作战卡 | Crayon/Klue | 竞品威胁、客户问题、我方回应、证明材料 | 已实现 |
| 产品线匹配 | 垂直化差异点 | EM4/EMX/E1/M1 Plus/P6 匹配 | 已实现 |

## 3. P1 能力补齐

P1 是增强专业度的能力，目标是让系统从“可用”进入“像真实产品”。

| 能力 | 借鉴对象 | 网页表现 | 当前实现 |
|---|---|---|---|
| 市场地图 | CB Insights | 行业格局分层：客户、竞品、场景、风险 | 本次新增 |
| 情报周报/月报 | Feedly/Meltwater | 自动摘要本轮重点机会、风险和动作 | 本次新增 |
| Win/Loss 复盘 | Klue | 胜负因素、客户异议、改进动作 | 本次新增 |
| 竞品趋势图 | Similarweb | 竞品动态热度和威胁等级 | 本次新增 |
| Roadmap 视图 | Aha! | 产品、研发、测试、售前动作分层 | 已实现并增强 |

## 4. P2 能力补齐

P2 是生产化方向，目标是说明这个系统如何进入真实企业协同。

| 能力 | 借鉴对象 | 网页表现 | 当前实现 |
|---|---|---|---|
| 内部知识库融合 | AlphaSense | RAG 知识库、资料来源、问答依据 | 已实现 |
| 多源告警 | Dataminr/Meltwater | RSS/搜索/API/网页源统一监控 | 已实现基础版 |

| 角色权限和审计 | Productboard/Aha! | 角色与审计规划 | 本次新增展示 |
| 供应商评分卡 | CB Insights | 供应商能力、风险、证据要求 | 本次新增 |

## 5. 前端改造

### 5.1 新增“能力路线图”页面

页面展示 P0/P1/P2 三组能力，每个能力包括：

1. 能力名称
2. 借鉴来源
3. 产品价值
4. 当前状态
5. 下一步动作

### 5.2 改造“决策中心”

新增：

1. P0/P1/P2 能力补齐概览
2. 市场地图
3. 情报简报
4. Win/Loss 复盘
5. 供应商评分卡入口

### 5.3 改造“竞品对比”

保留结构化对比，但不再让页面像数据库表格。参数表改为产品化卡片：

1. 产品定位
2. 技术路线
3. 感知能力
4. 车规与量产
5. 公开客户
6. 对我方影响

### 5.4 改造“配置/结果输出”

所有原始 JSON 输出改为业务化文案或卡片：

1. 配置状态卡片
2. 方案结果卡片
3. RAG 答案卡片
4. 外部协同结果卡片

## 6. 后端改造

`/api/decision-center` 新增字段：

```json
{
  "capability_roadmap": {
    "p0": [],
    "p1": [],
    "p2": []
  },
  "market_map": [],
  "briefing": {},
  "win_loss": [],
  "supplier_scorecards": []
}
```

所有字段均提供本地规则兜底，不依赖大模型才能展示。

## 7. 验收标准

1. 网页不出现 `parameter/value` 风格调试内容。
2. P0/P1/P2 在网页中完整可见。
3. 竞品对比不再只有表格，而是可读卡片。
4. 配置状态不再直接展示 JSON。
5. 方案、RAG、协同结果优先展示业务化结论。
6. 决策中心可以清楚说明当前产品已补齐哪些竞品能力、还有哪些生产化方向。
